# 图片旋转检测准确率测试方案

## 一、测试目标

对 rotation_dataset.json 中的 480 张图片进行批量测试，评估两个阶段的准确率：
1. **粗调准确率**：能否正确检测 90°/180°/270° 大角度旋转
2. **精调准确率**：在粗调之后，能否检测出 1-2° 的微小倾斜

## 二、代码架构

### 1. 核心函数复用
- `coarse_adjust_with_tesseract()`: 使用 Tesseract OCR 检测大角度旋转
- `calculate_fine_skew()`: 使用 OpenCV 形状检测计算微小倾斜角度

### 2. 角度分类逻辑

#### 角度标准化
将所有角度归一化到 [0, 360) 范围内

#### 分类规则
- **0°类**: 0-5° 或 355-360°
- **90°类**: 85-95°
- **180°类**: 175-185°
- **270°类**: 265-275°

**为什么用范围而不是精确值？**
因为数据集中的 angle 字段本身可能就是带有1-2度误差的标注值

## 三、粗调准确率测试

### 测试逻辑
```
对每张图片：
1. 下载图片
2. 使用 Tesseract OSD 检测旋转角度
3. 将检测角度和真实角度都分类到标准类别
4. 比较分类结果是否一致
```

### 准确率计算
```
准确率 = (正确预测数 / 成功测试总数) × 100%
```

**注意**：90° 和 270° 必须严格区分，检测结果必须与真实角度分类完全匹配才算正确。

## 四、精调准确率测试

### 测试策略（重要！）
**所有 480 张图片都参与精调测试**，而不是只测试小角度图片。

测试逻辑：
1. **如果粗调正确**：使用粗调检测的角度旋转图片，测试精调能力
2. **如果粗调错误**：使用真实角度的**归一化分类值**旋转图片，测试精调能力
   - ⚠️ 注意：不是直接用精确的真实角度（那样就没有精调的意义了）
   - 而是先对真实角度归一化分类，例如：
     - 真实角度 92° → 归一化为 90°类 → 旋转 90° → 剩余 2° 需要精调检测
     - 真实角度 358° → 归一化为 0°类 → 旋转 0° → 剩余 -2° 需要精调检测

这样做的好处：
- 可以评估算法在图片基本摆正后的微小角度检测能力
- 即使粗调失败，也能继续测试精调（使用归一化角度兜底）
- 能够分组统计：粗调正确时的精调表现 vs 粗调错误时的精调表现

### 测试逻辑
```
对每张图片：
1. 下载图片
2. 获取粗调结果（是否正确、检测的角度）
3. 决定旋转角度：
   - 粗调正确 → 使用粗调检测的角度
   - 粗调错误 → 使用真实角度的归一化分类值（不是精确值！）
4. 旋转图片到基本正确的方向
5. 使用 OpenCV 检测旋转后的微小倾斜角度
6. 计算精调误差 = |检测角度 - (真实角度 - 旋转角度)|
7. 误差 ≤ 2° 认为正确
```

### 角度计算示例
**示例1：真实角度 92°，粗调正确**
- 粗调检测为 90°
- 使用 90° 旋转图片
- 真实精调角度 = 92° - 90° = 2°
- 检测精调角度应该接近 2° 才算正确

**示例2：真实角度 92°，粗调错误（检测为180°）**
- 粗调失败，使用真实角度归一化：92° → 90°类
- 使用 90° 旋转图片（而不是92°！）
- 真实精调角度 = 92° - 90° = 2°
- 检测精调角度应该接近 2° 才算正确

这样无论粗调是否成功，都能测试到精调能力。

### 评估指标
1. **准确率**: 误差 ≤ 2° 的比例
2. **平均误差**: 所有测试图片的平均角度误差
3. **中位数误差**: 更鲁棒的误差指标，不受极端值影响

## 五、结果输出

### 文件结构
```
test_results/
├── coarse_accuracy_results.json    # 粗调详细结果
├── fine_accuracy_results.json      # 精调详细结果
└── summary.json                     # 汇总报告
```

### 详细结果字段
每条测试记录包含：
- `sourceFile`: 来源文件名
- `imageUrl`: 图片 URL
- `true_angle`: 真实角度
- `detected_angle`: 检测角度
- `is_correct`: 是否正确
- `status`: 测试状态（成功/失败/错误）

### 汇总指标
- 总图片数
- 成功测试数
- 正确预测数
- 准确率
- 平均误差（仅精调）
- 中位数误差（仅精调）

python test_accuracy.py

## 八、优化建议

### 如果准确率不理想，可以尝试：

1. **粗调优化**
   - 调整 Tesseract 的配置参数
   - 尝试其他 OCR 方向检测方法

2. **精调优化**
   - 调整形态学操作的核大小（目前是 30x1）
   - 调整轮廓面积阈值（目前是 500）
   - 尝试霍夫变换等其他直线检测方法

3. **数据预处理**
   - 增加图片对比度增强
   - 二值化阈值自适应调整
   - 去噪声处理
