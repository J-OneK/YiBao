# 角度方向测试验证

## 各组件角度定义总结：

### 1. 数据集 angle 字段
- `angle: 90` = 图片被**顺时针旋转90度**

### 2. Tesseract OSD
- `Rotate: 90` = 需要**顺时针旋转90度**才能摆正
- 转换：`detected_angle = 360 - tesseract_angle`（90/270互换）

### 3. PIL rotate()
- `rotate(正数)` = **逆时针**旋转  
- `rotate(负数)` = **顺时针**旋转

### 4. OpenCV minAreaRect
- 返回的angle范围：**[-90, 0)**
- 经过我们的处理后范围：**[-45, 45]**
- **负数** = 图片逆时针倾斜
- **正数** = 图片顺时针倾斜

### 5. 我们的精调返回值定义
- 返回值 = 图片被**顺时针旋转**的角度
- 正数：顺时针旋转
- 负数：逆时针旋转

## 完整流程验证：

### 场景1：图片被顺时针旋转17度

1. **数据集**：`angle: 17`
2. **分类**：`angle_to_category(17) = 0`（在0-5度范围外，但接近0）
   - 实际上17度不在任何标准分类中，会返回17
3. **粗调旋转**：`rotate(17)` = 逆时针旋转17度 ✅ 摆正
4. **精调检测**：
   - 图片已摆正，应该检测到 **0度**
   - OpenCV检测到的倾斜应该是0
5. **精调角度计算**：
   - `true_fine_angle = 17 - 17 = 0` ✅
   - `detected_fine_angle` 应该 ≈ 0

### 问题图片分析：`0334_true17_rot17_fine0.0_detected34.jpg`

- true17：真实角度17度
- rot17：用17度旋转矫正（这里有问题！）
- fine0.0：真实精调应该是0度 ✅
- **detected34**：检测到34度 ❌ 大错特错！

**可能的原因**：
1. ❌ `angle_to_category(17)` 返回了17（不在标准分类中）
2. ❌ 用17度旋转是错的，应该用0度（因为17度不是90的倍数）
3. ✅ 问题在于：**17度的图片根本不应该被旋转**

## 修复方案：

### 问题：
`angle_to_category(17)` 返回17，但这不是标准的粗调角度！

### 解决：
对于不在标准分类中的角度，应该：
- 如果粗调错误，使用 **最接近的标准角度**（0/90/180/270）
- 而不是使用原始角度
